Detailed Description

Deals with objects stored in a priority queue EG a drawable is stored in a priority queue, used to decide the order in which drawables are drawn to the screen For now, priority queue is used to implement a timer (as usual, this could be done with less overhead...)

a Pool can contain objects in many priority Queues, but a Queue can contain only objects from one Pool
